# Fine-tuning Memory

このディレクトリは、ファインチューニングに関する知見をリポジトリ内に残すための Memory です。

対象は、成功パターンだけではありません。注意点、失敗、うまくいかなかった仮説、環境依存の詰まり、評価で見つかった問題も残します。後から読む人が同じ失敗を避け、なぜその判断をしたのか追えることを重視します。

## 何を残すか

- データセット作成で分かったこと
- JSONL や chat template の形式ミス
- 学習が失敗した原因と対処
- 小さい PoC でうまくいった設定
- LoRA、DPO、量子化、評価に関する注意点
- モデルが覚えるべきことと RAG に任せるべきことの切り分け
- 初学者が誤解しやすい概念の説明

## 書き方

- `entries/` に Markdown で1件ずつ追加する。
- ファイル名は `YYYY-MM-DD-short-title.md` にする。
- 1つのメモは1つの主題に絞る。
- 初学者にも分かるように、専門用語は短く説明する。
- 成功だけでなく、失敗と判断理由も残す。
- 実データ、秘密情報、巨大なログ、モデル重みは書かない。

## Template

新しい知見は `templates/knowledge-note.md` をコピーして書く。
